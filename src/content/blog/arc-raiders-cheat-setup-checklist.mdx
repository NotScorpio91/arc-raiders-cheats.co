The fastest way to waste a fresh Arc Raiders cheat purchase is to skip prep and queue a ranked extraction ten minutes after download. Loaders are picky about environment. Overlays behave differently after patches. OBS can leak a box you forgot was enabled. This checklist is the sequence we recommend before any live session — and again after every update.

Use it with the tier you bought on [cheats](/cheats/), the support products you actually need from [products](/products/), and honest expectations from our [EAC guide](/blog/arc-raiders-eac-and-cheats/). No step here promises safety. Every step reduces amateur failure.

## Phase 1 — Before You Touch the Loader

### System and account prep

- [ ] Confirm your tier — [Xray](/cheats/xray/), [Pro](/cheats/pro/), or [Private](/cheats/private/) — matches the features you plan to enable day one.
- [ ] Close overlapping overlays: RTSS, Discord overlays, legacy mapping tools, other game injectors.
- [ ] Update GPU drivers only from official sources — avoid random "optimizer" bundles the same day as cheat install.
- [ ] Note your current Arc Raiders game build and any pending patch. If a update dropped today, check provider status before launching.
- [ ] If you are recovering from a hardware flag, complete HWID prep first — see [spoofer guide](/blog/arc-raiders-spoofer-guide/) and [HWID Spoofer](/products/hwid-spoofer/).

### Files and access

- [ ] Download from your delivered link only — not mirrors, not re-uploaded Discord attachments.
- [ ] Extract to a simple path without special characters if the loader docs recommend it.
- [ ] Run a quick AV exception only if your provider explicitly instructs it. Random exclusions for unknown binaries are a security risk.

<BlogCallout variant="warning" title="Patch day rule">
  If Arc Raiders or the loader posted a downtime notice in the last few hours, stop here. Playing through broken status is how fresh accounts die on day one.
</BlogCallout>

## Phase 2 — Loader Install and Launch

### First boot

- [ ] Run the loader as administrator **only** if documentation requires it.
- [ ] Log in or activate license per delivery instructions.
- [ ] Enable capture exclusion now if you stream — details in [stream-safe overlays](/blog/arc-raiders-stream-safe-overlays/).
- [ ] Confirm the loader sees the correct Arc Raiders install path.
- [ ] Inject or attach using the provider's current method — do not mix old YouTube steps from prior seasons.

### Failure handling

If injection fails:

1. Reboot once — boring, effective.
2. Verify game integrity through your launcher.
3. Remove newly installed conflicting software.
4. Contact support with exact error text, not "it broke."

Screenshot errors. Guessing adds hours.

## Phase 3 — Overlay and Menu Configuration

### Baseline ESP profile

Build a `baseline` profile before you fantasize about max settings:

- [ ] Player ESP on with distance cap suited to your loadout.
- [ ] Loot ESP filtered — value or category, not everything.
- [ ] Skeletons off until you need them for ARC weak points.
- [ ] Health bars and distance readouts on if your tier includes them.
- [ ] Save the profile with a clear name.

### Combat features — optional on day one

- [ ] Aim smoothing and FOV inside conservative ranges — see [low-profile settings](/blog/arc-raiders-cheat-settings-low-profile/).
- [ ] Trigger tools off until you understand cover behavior.
- [ ] Recoil support low enough that spray patterns still look human.

### Private tier extras

[Private](/cheats/private/) users get more categories. That means more ways to leave noise on. Duplicate the baseline profile into `farm` and `pvp` variants instead of one mega config.

## Phase 4 — Verification Before Live Queue

### Visual check

- [ ] Toggle each ESP category and confirm readability — no full-screen clutter.
- [ ] Verify FOV rings or combat widgets are off unless required.
- [ ] Record thirty seconds of local gameplay or OBS capture.
- [ ] Scrub recording for leaked overlays if you broadcast.

### Behavior check

- [ ] Dry-fire aim settings on ARC targets at realistic engagement ranges.
- [ ] Test loot route on a low-stakes run — extract early, do not hero push.
- [ ] Watch your own clip once. If you cringe at a snap, tune before stacking high-tier gear.

<BlogCallout variant="tip" title="First session goal">
  Session one is for verification, not domination. One clean extract with quiet settings beats three flashy deaths that train bad habits.
</BlogCallout>

## Phase 5 — First Live Raid

### Queue discipline

- [ ] Pick a map and loadout you already know without software.
- [ ] Set a session timer — ninety minutes is plenty for day one.
- [ ] Disable experimental features you have not tested.
- [ ] Communicate with squadmates if you are duo — sudden callouts that do not match camera look suspicious.

### During raid

- Rotate like you would without ESP — software informs routes; it should not autopilot them.
- Take fights you would take without aim assist, just with tighter execution.
- Extract with loot instead of chasing revenge kills to test max FOV.

### After raid

- [ ] Note loader version, game build, and settings used.
- [ ] Save any tweaks as a new profile — do not overwrite baseline until you have a week of stability.
- [ ] If anything felt off — stutter, ghost inputs, menu freeze — stop and diagnose before requeue.

## Phase 7 — After Updates

Re-run this shortened loop whenever Arc Raiders patches, EAC updates, or your loader revs:

1. Check provider status — do not launch if down.
2. Verify game files.
3. Boot loader and confirm version string matches announcement.
4. Load `baseline` profile — not yesterday's experimental PvP max.
5. One short test raid before long sessions.

Bookmark the [blog](/blog/) for new setup guides as seasons change.

## Common Checklist Failures

**Skipped conflict cleanup** — two overlays fight, loader blames anti-cheat, user blames seller. Remove one variable at a time until the loader boots clean, then add features back slowly.

**Max settings day one** — reports arrive before you learn the map meta. Season 1 routes and extract timings still matter; software does not replace map literacy.

**No local recording test** — stream leaks discovered by chat, not by you. Thirty seconds of OBS capture costs less than one ban appeal afternoon.

**Ignored downtime** — undetected on the store page does not mean safe during an active patch. Provider status pages exist for a reason.

**Same machine, new account, no HWID plan** — hardware-linked flags repeat. Pair account resets with the [spoofer guide](/blog/arc-raiders-spoofer-guide/) when the situation calls for it.

**Stacked products before baseline works** — DMA, spoofers, and premium tiers all at once make every failure harder to diagnose. Sequence matters.
